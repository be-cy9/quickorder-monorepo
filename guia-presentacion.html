Comunicación entre Microservicios

IE 2.4.1 · IE 2.4.2
📡 Flujo completo: GET /api/pedidos/cliente/1
1
Postman llama al API Gateway
GET http://localhost:8080/api/pedidos/cliente/1
El Gateway lee la ruta /api/pedidos/** → enruta a 8081
2
PedidoController recibe la petición
PedidoController.java → getPedidosConCliente(clienteId=1)
Delega al PedidoService — el Controller no tiene lógica de negocio.
3
PedidoService llama a ClienteService
PedidoService.java → clienteService.obtenerClientePorId(1)
PedidoService necesita datos del cliente → delega a ClienteService.
4
ClienteService hace llamada HTTP a ms-clientes
ClienteService.java → llamarMsClientes(1)
GET http://localhost:8082/api/clientes/1
WebClient ejecuta la petición HTTP. La respuesta JSON se deserializa automáticamente en un ClienteDTO.
5
PedidoService consulta su propia BD
PedidoRepository.findByClienteId(1) → BD local db_pedidos
Obtiene los pedidos del cliente desde la base de datos propia de ms-pedidos.
6
Respuesta combinada de dos microservicios
new RespuestaCombinadaDTO(cliente, pedidos)
Un solo JSON con datos de ms-clientes y de ms-pedidos combinados.

service/ClienteService.java 
public ClienteDTO obtenerClientePorId(Long id) {
    try {
        return llamarMsClientes(id); // ← HTTP
    } catch (Exception e) {
        // FALLBACK GRACEFUL
        ClienteDTO def = new ClienteDTO();
        def.setNombre("Cliente no encontrado");
        return def;
    }
}
protected ClienteDTO llamarMsClientes(Long id) {
    return webClient.get()
        .uri("http://localhost:8082/api/clientes/{id}", id)
        .retrieve()
        .bodyToMono(ClienteDTO.class)
        .block();
}

Reglas de Negocio — PedidoService

IE 2.2.1 · IE 2.2.2 · IE 2.1.4

📄 PedidoService.java — guardarPedido()

// Estados válidos definidos como Set (búsqueda O(1))
private static final Set<String> ESTADOS_VALIDOS = Set.of(
    "PENDIENTE", "PROCESADO", "ENVIADO", "ENTREGADO", "CANCELADO"
);

public Pedido guardarPedido(Pedido pedido) {
    // ① REGLA 1: validar estado antes que todo
    if (!ESTADOS_VALIDOS.contains(pedido.getEstado().toUpperCase())) {
        throw new BadRequestException("Estado inválido: ...");
    }
    // ② REGLA 2: validar que el cliente existe (llama a ms-clientes)
    ClienteDTO cliente = clienteService.obtenerClientePorId(pedido.getClienteId());
    if (cliente == null || cliente.getId() == null) {
        throw new BadRequestException("Cliente no válido");
    }
    // ③ Normalizar + guardar
    pedido.setEstado(pedido.getEstado().toUpperCase()); // "pendiente" → "PENDIENTE"
    pedido.setFechaCreacion(LocalDateTime.now());
    return pedidoRepository.save(pedido);
}

@RestControllerAdvice // intercepta TODOS los controllers
public class GlobalExceptionHandler {

    private static final Logger log = ...;

    @ExceptionHandler(ResourceNotFoundException.class)
    public ResponseEntity handleNotFound(...) {
        log.warn("[404 NOT FOUND] en {}: {}", path, msg);
        // → JSON: {status:404, error, message, path}
    }

    @ExceptionHandler(BadRequestException.class)
    public ResponseEntity handleBadRequest(...) {
        log.warn("[400 BAD REQUEST] en {}: {}", path, msg);
    }

    @ExceptionHandler(Exception.class)
    public ResponseEntity handleAll(...) {
        log.error("[500 ERROR] en {}: {}", path, msg, ex);
    }
}

📐 Estructura de un test — Patrón AAA

@Test
@DisplayName("guardarPedido - Camino Feliz")
void guardarPedido_debeGuardar_cuandoClienteValido() {
  // ARRANGE: configurar mocks
  when(clienteService.obtenerClientePorId(1L))
    .thenReturn(clienteEjemplo);
  when(pedidoRepository.save(any()))
    .thenReturn(pedidoEjemplo);

  // ACT: ejecutar el método real
  Pedido resultado = pedidoService
    .guardarPedido(pedidoEjemplo);

  // ASSERT: verificar resultado
  assertNotNull(resultado);
  verify(pedidoRepository, times(1)).save(any());
}

💡 Conceptos clave para explicar

  • @Mock: objeto totalmente simulado — no toca BD ni llama servicios reales
  • @InjectMocks: crea el objeto real e inyecta los @Mock en su constructor
  • @Spy: objeto real pero permite interceptar métodos específicos
  • when().thenReturn(): define qué devuelve el mock cuando se llama
  • verify(): verifica que el método fue llamado exactamente N veces

Preguntas del Profe — Guía de respuestas

❓ ¿Qué es el patrón CSR?
Controller recibe la petición HTTP y delega → Service tiene la lógica de negocio y las reglas → Repository accede a los datos. Cada capa tiene una sola responsabilidad.
❓ ¿Por qué WebClient y no RestTemplate?
WebClient es el cliente HTTP moderno de Spring 5+. RestTemplate está en modo mantenimiento (deprecated). WebClient soporta programación reactiva y es más eficiente.
❓ ¿Qué hace .block()?
Convierte la llamada reactiva (asíncrona) en síncrona. La usamos porque el resto del microservicio usa Spring MVC (no WebFlux reactivo completo). En un sistema 100% reactivo no lo necesitaríamos.
❓ ¿Qué es un DTO?
Data Transfer Object: objeto simple para transferir datos entre capas o microservicios, sin exponer directamente la entidad JPA. ClienteDTO no tiene anotaciones @Entity — solo datos.
❓ ¿Por qué ms-pedidos tiene su propia copia de ClienteDTO?
Principio de autonomía de microservicios: cada microservicio es independiente. Si ms-clientes cambia su modelo Cliente, ms-pedidos no se rompe porque tiene su propia copia del DTO.
❓ ¿Qué pasa si ms-clientes se cae?
Aplicamos el patrón fallback graceful: el catch(Exception e) en ClienteService captura el error y devuelve un ClienteDTO con datos por defecto. El sistema sigue operativo, no lanzamos error al usuario.
❓ ¿Qué es @RestControllerAdvice?
Es un interceptor global de excepciones: captura los errores lanzados en cualquier controller y los convierte en respuestas HTTP estructuradas (JSON con timestamp, status, message, path). Evita duplicar el manejo de errores.
❓ ¿Por qué Set.of() para los estados válidos?
Set tiene búsqueda O(1) — constante, sin importar cuántos estados haya. contains() en un Set es mucho más eficiente que una cadena de if/else o switch.
❓ ¿Cómo funciona el API Gateway?
Lee la predicates → Path del application.yml: si la petición llega a /api/pedidos/**, la reenvía al uri: http://localhost:8081. El cliente externo siempre habla con el puerto 8080.
❓ ¿Qué diferencia hay entre @Mock y @Spy en los tests?
@Mock crea un objeto totalmente falso — ningún método hace nada real. @Spy crea el objeto real pero permite interceptar métodos específicos con doReturn().
❓ ¿Por qué JpaRepository y no escribir SQL directo?
Spring Data JPA genera el SQL automáticamente desde el nombre del método. findByClienteId(Long id) genera SELECT * FROM pedidos WHERE cliente_id = ?. Menos código, menos errores, más mantenible.
